# AgentMem-Flow M0 分析报告

## 版本信息

- 大版本：M0 - 验证与基线层
- 合并旧版本：V0、V1、V2
- 对应计划：建立可复现实验闭环、精确真值、朴素 SSD 图检索基线、页面布局优化基线
- 状态：已完成

## 合并范围

M0 合并了原始报告中的前三个基础阶段：

- V0：内存精确向量检索基线，用暴力 Top-K 建立 Recall@10 的真值对照。
- V1：朴素 SSD 图检索基线，将图遍历映射到磁盘页访问，暴露随机 I/O 放大。
- V2：Co-Access Packed Page Layout，通过随机、BFS、共访问三种布局比较页面局部性。

这一层的目标不是追求最终性能，而是让后续所有优化都有可解释的参照系：同一数据集、同一查询、同一真值、同一统计口径。

## 核心能力

### 精确真值与评测闭环

V0 引入 `VectorSet`、`.fvecs/.ivecs` 数据加载、Synthetic 数据集与暴力精确搜索，用于计算 Recall@10、查询延迟、吞吐等核心指标。该阶段解决了评测不可复现和缺少真值的问题。

### 朴素磁盘图检索

V1 引入 `--engine graph` 路径，将向量图以一节点一页的方式放到磁盘文件中，通过 ANN 图遍历模拟 SSD 随机读。该实现确认了 Page Cache 友好假设在图搜索中并不成立，随机读成为主要瓶颈。

### 页面布局优化基线

V2 引入随机布局、BFS 布局和共访问布局。共访问布局根据历史访问邻接关系将高共现节点打包到同一页，显著降低每次查询触发的磁盘页读取数量。

## 指标摘要

| 阶段 | 关键实验 | 结果 | 结论 |
| --- | --- | --- | --- |
| V0 | 精确暴力搜索 | 提供 Recall@10 真值 | 后续 ANN 结果可被稳定验证 |
| V1 | 朴素 SSD 图遍历 | Avg SSD Reads/Query 约 176.6750 | 图搜索产生严重随机读放大 |
| V2 | Co-Access Packed Layout | Avg SSD Reads/Query 降至约 21.4100 | 布局局部性是第一层有效 I/O 优化 |

## 计划问题映射

- P0：缺少向量检索基线，已由 V0 解决。
- P1：缺少真实数据与真值加载，已由 V0 解决。
- P2：缺少 SSD 图检索基线，已由 V1 解决。
- P3：随机 I/O 放大问题，由 V1 暴露并由 V2 初步缓解。
- P8/P9：自动化与可复现实验框架，由 V0-V2 建立雏形。

## 阶段结论

M0 证明了项目的基本技术路线是必要的：在图索引外存化后，默认文件系统缓存和朴素布局无法支撑高频随机访问；仅靠算法层搜索参数无法解释 I/O 行为，必须同时记录页面读取、布局策略、Recall 与延迟。

M0 产出的最重要资产是“可复现实验闭环”。后续 M1-M4 的缓存、预取、写入、构图和最终评测都必须继续复用这一套真值与指标口径。
